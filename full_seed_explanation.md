# Understanding `full_seed.sql`

This document briefly explains what `full_seed.sql` is and how it fits into the project's data seeding workflow.

## 1. What is `full_seed.sql`?
`full_seed.sql` is a SQL script containing a database dump or manually constructed seed data intended for a full database reset. 
Based on its header (`-- Generated: 2026-02-18T20:11:20.553Z`), it was likely exported from a database management tool (like Supabase dashboard, pgAdmin, or DBeaver) or generated by an external script not present in this repository.

The file performs three main operations:
1. **Schema Updates:** Ensures necessary columns exist (`address`, `last_verified`, `zone`, `target_audience`, `google_maps_link`).
2. **Clean Slate:** Runs `TRUNCATE TABLE public.spots;` to wipe existing data.
3. **Insert Data:** Bulk inserts 268 records into the `spots` table in a single `INSERT INTO` statement.

## 2. Database Schema (`public.spots`)

The following table describes the columns in the `spots` table and how they are used:

## 3. How it is Used in the Project
`full_seed.sql` acts as the **raw input** for the Python script `update_sql.py`. It is not meant to be run directly on the production database in its raw form.

### The Processing Pipeline (`update_sql.py`)
The `update_sql.py` script automatically processes `full_seed.sql` to generate a final executable SQL file (`updated_seed.sql`). It performs the following enhancements:

- **Data Parsing:** Parses the raw SQL values from `full_seed.sql` safely.
- **Data Imputation (City Inference):** Automatically infers the `city` based on the `location_id` ranges (e.g., `< 1000` is Chennai, `3000-4000` is Bengaluru, etc.) if the city is missing (`NULL`).
- **Google Maps Link Generation:** Dynamically constructs and URL-encodes a Google Maps search link (`https://www.google.com/maps/search/?api=1&query=...`) using the venue name, address, area, and city. It appends this generated link to each record.
- **Chunking:** Splits the single massive `INSERT` statement into smaller chunks of 50 records per statement. This prevents statement size limits when executing the SQL against the database.
- **Transaction Safety:** Wraps the entire operation in a `BEGIN;` and `COMMIT;` block to ensure that the database reset is atomic (either all data imports successfully, or none of it does).

## 3. Workflow Summary
To update the seed data:
1. Export or manually update `full_seed.sql` with new raw data.
2. Run `python update_sql.py`.
3. The script reads `full_seed.sql` and generates `updated_seed.sql`.
4. Execute `updated_seed.sql` on your database to safely reset and seed the tables with the enhanced data.
